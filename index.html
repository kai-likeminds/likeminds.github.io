release notes

The release notes summarize the changes in the current and previous product features. The Authenion enables a secure Single Sign-On (SSO) solution for Oracle proprietary applications. The product uses PingFederate (or) any other standard OpenID Connect provider as Identity Provider for authentication related events

system reuirement

1 Supported Operating Systems ▪ Red Hat Enterprise Linux 7 & above ▪ CentOS 7 & above ▪ Oracle Enterprise Linux 7 & above 2.2 Linux User requirements in the Authenion Server ▪ Dedicated Linux user in the name of ‘appuser’ needs to be created to execute Authenion scripts. ▪ The Linux user should be granted with Read/Write privileges for the Authenion deployment package. The Authenion should be installed in the ‘/home/appuser’ directory. 2.3 Supported Java environment ▪ OpenJDK 11 ▪ Oracle Java SE Development Kit 11 LTS / Oracle Java SE Runtime Environment (Server JRE) 8 2.4 Minimum Hardware requirements ▪ 2 CPU/Cores recommended ▪ 2 GB of RAM ▪ 15 GB available space 2.5 Recommended Hardware requirements ▪ 4 CPU/Cores recommended ▪ 4 GB of RAM ▪ 30 GB available space

getting started

2 Enterprise Deployment Model Authenion supports a Gateway architecture for its protected Oracle applications. In this model, the traffic is initially directed to Authenion. Authenion grants or denies access by redirecting the traffic to Authentication providers. ▪ This deployment model can be set up quickly without disrupting the applications. The Authenion will not be installed on separate servers from the application web servers. ▪ With the fewer component architecture, it is easy to identify issues and troubleshoot. All transactions are logged, and it is easier to view any specific events.

3 What can I do with Authenion? Authenion is a light-weight product that allows you to enable a secure Single Sign-On solution for your Oracle Enterprise applications. The main functionalities: ▪ Use Authenion to protect Oracle applications ▪ Connect Authenion with external OpenID Connect providers for authentication ▪ You can manage sessions for the protected applications and use OAuth scopes to send user data to applications 3.4 How does Authenion work? Authenion acts as the gateway to your Oracle Enterprise Applications and capture user requests & delegate it to your Identity Provider (IdP) for authentication. Session Initiation ▪ Using advanced & secured integration standards, Authenion securely connects with your Identity Provider using OAuth/OIDC protocols. This allows Authenion protected Oracle Applications requests to be delegated to your centralized Identity Provider in your organization. ▪ The Identity Provider will be responsible for challenging the user requests with strong authentication schemes. The Identity Provider will also be responsible for collecting the user attributes and send it to Authenion through OAuth/OIDC tokens for further processing such as session establishment & granting user-access.

  1. User request access for Oracle Applications
  2. Authenion checks for the user session, and it redirects the user’s request to IdP for authentication
  3. IdP authenticates the user using the adapters and against a data store configured to it
  4. Authenion will receive the authenticated user’s attributes and parse the attributes into HTTP-Headers and send it to the target application
  5. IdP authenticates the user using the adapters and against a data store configured to it
  6. Oracle Applications will retrieve & validate the HTTP-Headers from Authenion and grant access to users based on their roles and responsibilities
